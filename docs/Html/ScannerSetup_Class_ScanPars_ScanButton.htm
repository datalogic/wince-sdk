Scan Button

 

This parameter enables/ disables the association between the reading function (barcode/rfid/imager/....) and the SCAN button.

 

That means that, if it is enabled, when the user presses the SCAN button the reading function is started, whereas when the user releases the SCAN button the reading function is blocked.

 

 

Syntax

Name:

SCAN_PARAM_SCAN_BUTTON

Set by:

Value

Values:

SCAN_PARAM_DISABLE

SCAN_PARAM_ENABLE

Default:

SCAN_PARAM_ENABLE

 

 

Requirements

Header:

declared in SCAN_PARAMETERS.H, include SE_PARAMETERS.H.

 

NOTE

if the user wants to monitor the pressing and releasing of the SCAN button from the application program, he needs to monitor two 'named events' defined in se_sid.h:
SED_SCAN_BUTTON_PRESS_EVENT_NAME, SED_SCAN_BUTTON_RELEASE_EVENT_NAME through the function WaitForSingleObject or the function WaitForMultipleObjects

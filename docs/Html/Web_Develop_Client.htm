Web Client

 

NOTE

This feature is not supported by the Datalogic Skorpio™ batch models using WinCE 5.0 Core.

 

 

Your Datalogic mobile computer or PDA can be used as a Web Client communicating with a Web Server to navigate web applications.

The web browser used by the mobile computer is Microsoft® Internet Explorer 6.0, which provides the following functions (for further details refer to Microsoft® documentation):

- complete ActiveX Hosting support, which allows creating HTML pages including the Datalogic ActiveX Controls;

- Internet Explorer 6.0 DOM;

- Dynamic HTML;

- XML Parser;

- JavaScript Parser 5.5.

 

When the mobile computer requires a web page, its onboard web browser sends the server the HTTP_USER_AGENT header containing the string identifying the mobile computer itself.

Thanks to this information, the web site (server) recognizes the Datalogic mobile computer and is able to send it those web pages which are more suitable for the mobile computer functions (i.e. pages with JavaScript scripting for barcode reading) and its screen resolution.

 

For example, if using the ASP technology with JavaScript scripting language, the web server is able to get this information through the following instruction:

<% var sUserAgent = Request.ServerVariables(“HTTP_USER_AGENT”); %>

Then, the contents of the sUserAgent variable string will be the following:

Mozilla/4.0 (compatible; MSIE 6.0; Windows CE; Device_Name; 240x320)

 

where Device_Name is one of the following Datalogic devices: Viper-NET, Kyman-NET, Rhino-NET, Datalogic Skorpio, Datalogic Skorpio-GUN, Datalogic Jet, Datalogic Memor.

 

 

If using the PHP technology, it is possible to get the same information by simply requiring the value of the $HTTP_USER_AGENT predefined variable.

 

The same information can be obtained in an HTML page by simply requiring the value of JavaScript navigator.userAgent property.

 

Thanks to the Internet Explorer 6.0 DOM, Document Object Model, it will also be possible to recognize the mobile computer screen resolution by requiring the value of the screen.width and screen.height properties.

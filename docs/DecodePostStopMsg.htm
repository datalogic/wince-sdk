DecodePostStopMsg

 

These messages occur at a system level of visibility, but are separate for each decoding device.  For each decoding device, only one message can be in effect at any time.  

 

BOOL DecodePostStopMsg(HANDLE hFile, HWND hWnd, UINT uMsg);

 

Parameters:

HANDLE hFile

Handle to the decoding device from which a scanning stop notification is requested. The CreateFile function returns this handle.

HWND hWnd

Handle to a window to receive a message when scanning stops.  If both hWnd and uMsg are set to NULL, the notification is cancelled (see Remarks section).

UINT uMsg

Message to send to the window when scanning stops.

Returns:

A value of TRUE is returned on success, otherwise FALSE is returned to indicate failure.

Remarks:

A message notification cannot be requested if an event has already been requested.  Setting both hWnd and uMsg to NULL can be used to cancel a previous message notification request, but only if the same file handle is used to both request and cancel the notification.

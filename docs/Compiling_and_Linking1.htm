Compiling and Linking

The C SDK is made up of Win32 style function calls, which are accessed using header files (.h) and library files (.lib).

Header files

The C SDK provides the following header files:

beaconAPI.h - Bluetooth beacon API

commonApi.h -common constant #define values

decodeApi.h (Decode API) - core Decoding API definitions and functions

decodeParam.h (Decode API) - enumerations and #defines provided for backwards compatibility

deviceApi.h (Device API) -core Device API definitions and functions

dlbt_api.h (Bluetooth API) - Bluetooth API definitions and functions

dlkeys.h -remappable key definitions

JoyaCradleApi.h (Joya Cradle API) - Joya Cradle API definitions and functions

SoftSpotApi.h (SoftSpot API) - SoftSpot API definitions and functions

wifiApi.h (Wi-Fi API) - Wi-Fi adapter API definitions and functions

 

Library files

You must also link to library file(s) to compile your application. Two options are provided:

Option 1 - link using the provided static library (Recommended)

This has the following advantages:

  • You only need to reference a single file to use all APIs.

  • The static library will first verify that a desired function is implemented on a target device before calling it.

  • If you attempt to call an unsupported function, an error code will be returned and GetLastError() will return ERROR_FUNC_NOT_FOUND.

  • It is useful if you are writing an application that will be compiled and executed on a variety of Datalogic devices.

To use the static library in your project, do the following:

  1. In your Visual Studio C++ project, add DatalogicAPI.lib to your list of import libraries.

  2. One way to accomplish this would be to right click on your project and click Properties. Click Configuration Properties > Linker > Input and enter DatalogicAPI.lib in the "Additional Dependencies" field. Using this method, make sure Configuration Properties > Linker > General > Additional Library Directories also points to the path containing DatalogicAPI.lib.

Option 2 - link using the provided import libraries

This option was the only option previously provided. It is a good choice for:

  • Existing projects only targeting a Datalogic device running Windows Mobile 6 or Windows CE 6.

To use the import library in your project, do the following:

1. In your Visual Studio C++ project, add the appropriate .lib file(s) to your list of import libraries:

  • decodeApi.lib = Decode API

  • DeviceApi.lib = Device API, Bluetooth API

  • JoyaCradleApi.lib = Joya Cradle API

  • SoftSpotApi.lib = SoftSpot API

  • wifiApi.lib = Wi-Fi API

  1.  One way to accomplish this would be to right click on your project and click Properties. Click Configuration Properties > Linker > Input and enter the required .lib file(s) in the "Additional Dependencies" field. Using this method, make sure Configuration Properties > Linker > General > Additional Library Directories also points to the path containing the .lib file(s).

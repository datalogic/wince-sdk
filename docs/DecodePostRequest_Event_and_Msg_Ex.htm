DecodePostRequestEventEx

 

This posts a request for data from a decoding device. The function will return immediately, and signal the specified events when the request is fulfilled or a timeout occurs.

 

DWORD cdecl DecodePostRequestEventEx(HANDLE hFile, DWORD reqType, HANDLE hEventRead, HANDLE hEventTimeout);

 

Parameters:

HANDLE hFile

DWORD reqType

Type of request being posted. It is either a number of samples to collect or an OR'ing of any of the values listed in Request Types (LINK). This ORâ€™ing is made up of the Number of data samples to collect (1-255) with whether it is a recurring request. For a recurring event the number of samples is ignored.

HANDLE hEventRead

Handle to an event to signal when the request is fulfilled with data.

HANDLE hEventTimeout

Handle to an event to signal when the request has timed out. If set to NULL, no event will be signaled when a timeout occurs.

 

Returns:

On success, a request ID is returned. On error, -1 is returned.

 

Remarks:

This should be used in a subsequent call to DecodeReadString and/or DecodeCancelRequest.

All notifications for posted requests are managed with a FIFO and reported in the order they take place.  This also means that notifications can occur later than the actual event if the requesting application does not process them as they happen.  The following example illustrates this:

1) DecodePostRequestEventEx is called.

2) DecodeSoftTrigger is called, and a bar code label is read.  (This is trigger attempt #1.)

3) The decode read event is signalled.

4) Before DecodeReadString is called, DecodeSoftTrigger is called again and a timeout occurs.  (This is trigger attempt #2.)

5) DecodeReadString is called and the label data is returned.  This relates to trigger attempt #1 (even though a second trigger attempt has been made).

6) The timeout event is signalled, now that the label data from trigger attempt #1 has been read.

7) The timeout event is handled, which relates to trigger attempt #2.

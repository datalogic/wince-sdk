DecodeReadStringEx

 

This reads data that was used to fulfill a request previously posted to a decoding device.

 

BOOL cdecl DecodeReadStringEx(HANDLE hFile, DWORD reqID, DWORD *stringLen, LPWSTR lpString, DWORD nMaxLen, CODE_ID *codeID, DWORD *idMod)

 

Parameters:

hFile

Handle to the decoding device to read data from. The CreateFile function returns this handle.

reqID

Request identifier of the request that has been satisfied with data to be read. The DecodePostRequest unctions return this identifier.

stringLen

The number of characters copied into the lpString buffer, not including the null terminator, is returned in the DWORD pointed to by stringLen.  This value is meaningful only success of the function. If NULL is passed into this parameter, then it is ignored.

lpString  

Pointer to a buffer to receive the text data read by the decoding device. The buffer must be large enough to store the entire string including the null terminator.

maxLen  

Maximum length, in characters, of the string that may be copied into the buffer pointed to by lpString (not including the null terminator).

codeID  

Pointer to a variable which will receive the code identifier for the string. NULL may be passed in as this parameter if the identifier is not of interest.

idMod

Pointer to a variable which will receive the AIM code ID modifier for the string. NULL may be passed in as this parameter if the modifier is not of interest.

Returns:

A value of TRUE is returned on success, otherwise FALSE is returned to indicate failure. Specifiying a request ID that has not yet been fulfilled will cause the function to return FALSE.

 

Remarks:

If the buffer is not large enough to contain the entire text data associated with the request ID, then the number of characters present in the text data is returned (not including a null terminator) into the DWORD pointed to by stringLen.

The value of maxLen is the length of the string that the buffer can hold, not the buffer size itself.  This is consistent with the minimum/maximum length parameters for a bar code symbology.  The buffer size should be large enough to hold a string of this length, plus a NUL terminator.  Specifically, buffer length in bytes = (stringLen + 1) * 2.

The text data returned will include symbology identifier characters, if enabled, appended to the beginning of the decoded data.  It does not include any label prefix or label suffix characters.  It is valid to have this function return a zero length string (stringLen), since null strings may be read by a decoding device.

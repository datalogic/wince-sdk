DecodeCancelRequest

 

This function cancels a request previously posted to a decoding device.

 

BOOL cdecl DecodeCancelRequest( HANDLE hFile, DWORD reqID );

 

Parameters:

HANDLE hFile

Handle to the decoding device that was posted to. The CreateFile function returns this handle.

DWORD reqID

Request identifier of the request to cancel. The DecodePostRequest functions return this identifier.

 

Returns:

A value of TRUE is returned on success, otherwise FALSE is returned to indicate failure.

 

Remarks:

Specifying a request ID that has not been posted will cause FALSE to be returned.

When this function is called, any notifications that were registered with the request when it was posted will be signalled.  This is done to allow threads waiting on these notification to be signalled.  Consider this example:

1) Thread A calls DecodePostRequestEvent with a handle to an event.

2) Thread B calls WaitForSingleObject with the event handle from step 1.

3) Thread A calls DecodeCancelRequest.  This will signal the event and allow thread B to run.

4) Thread B calls DecodeReadString which returns FALSE.  Calling GetLastError returns ERROR_INVALID_PARAMETER, because the request ID is no longer valid (it has been cancelled).

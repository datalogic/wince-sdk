Quality Attributes and Signature Conventions

 

Quality Attribute

This help file does not include standard functions such as Windows API.

Signature Conventions

a) Function Name is composed of <<category>+<functionality>> (e.g. “Bt” (for Bluetooth)+GetPairingInfo” (what it does)).

b) Return type

  1. All "Set" functions return a 'BOOL'. Errors return a value of 'FALSE'.

For example:

BOOL cdecl DispSetBkltBrightness(int Brightness);

  1. Simple "Get" functions which have only one quantity to return will return the data type of that value.

Errors return a value of '-1' by preference.

Exceptions:

  • If '-1' is a valid result for the return then use another defined value.

    • If result is a pointer to a buffer then use rule v) instead.

For example:

int cdecl BtGetStatus(void);

  1. Complex “Get” functions which have more than one return value return a ‘BOOL’. Errors return a value of ‘FALSE’.

For example:

BOOL cdecl BtGetPairingInfo(BTPAIR *pairs, DWORD pairCnt);

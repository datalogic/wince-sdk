Good Read Sound File

 

It defines the WAV file name played by the device on good read. When setting or getting this string, its binary format appears as follows:

 

 

- Setting String -

 

To set the string use setParameterByReference with SCAN_PARAM_GOOD_READ_SOUND_FILE as first argument.

The second argument of the setParameterByReference method must be a pointer to a 4-byte length field followed by a Unicode character array (2 bytes per character). The length field must contain the number of characters and not the number of bytes of the Unicode character array. The Unicode character array must contain the string the user wants to set as good read sound file.

For example, to set L”foo.wav” call:

 

SetParameterByReference(SCAN_PARAM_GOOD_READ_SOUND_FILE, Pointer);

- Getting String -

 

To get the string use getParameterByReference with SCAN_PARAM_GOOD_READ_SOUND_FILE as first argument.

The second argument of the getParameterByReference method must be a pointer to a 4-byte length field followed by an empty Unicode character array (2 bytes per character). The length field must be the capacity in terms of characters, and not bytes, of the following Unicode array. The method getParameterByReference copies the good read sound file string into the Unicode array, only if its length is ≤ than the length contained by the 4-byte length filed, while it always copies the string length into the 4-byte length field.

 

If the user needs to know the string length (without copying it), the length field must be set to 0 when calling the getParameterByReference method.

 

 

Syntax

Name:

SCAN_PARAM_GOOD_READ_SOUND_FILE

Set by:

Reference

Values:

see above

Default:

Beep.wav

 

 

Requirements

Header:

declared in SCAN_PARAMETERS.H, include SE_PARAMETERS.H.

 

 

NOTE

This parameter has no effect on Viper-NET™ and Rhino-NET™.

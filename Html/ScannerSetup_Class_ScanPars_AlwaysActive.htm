Scan Always Enabled

 

If enabled, the scanner is always ready to be activated by the trigger (HW and SW) and it is not necessary to use other applications (for ex., Barcode Reader applet) for reading barcodes.

If it is necessary to disable the driver after each reading attempt, the SCAN_PARAM_SCAN_ALWAYS_ENABLED parameter must be set to the SCAN_PARAM_SCAN_DISABLE_AFTER_READ_REQUEST value. In this way, after a reading attempt (i.e. when calling the DLScanner::scanEnable and DLScanner::pressSoftTrigger methods or the DLScanner::read one) the driver remains in stand-by mode and reading is disabled until the DLScanner::scanEnable or DLScanner::read method is called again. In case the user accidentally presses the scan button, this function prevents the driver from decoding new data while the application is still elaborating previous data.

 

 

NOTE

If both this parameter and the keyboard emulation parameter are enabled, the text of the read barcode is displayed by all Windows programs showing the characters corresponding to the key press on the keyboard.

 

 

Syntax

Name:

SCAN_PARAM_SCAN_ALWAYS_ENABLED

Set by:

Value

Values:

SCAN_PARAM_DISABLE

SCAN_PARAM_ENABLE

SCAN_PARAM_SCAN_DISABLE_AFTER_READ_REQUEST

Default:

SCAN_PARAM_DISABLE

 

 

Requirements

Header:

declared in SCAN_PARAMETERS.H, include SE_PARAMETERS.H.

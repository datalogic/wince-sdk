Preamble String

 

It defines the character string returned as a block before the barcode(s).

When setting or getting a preamble string, its binary format appears as follows:

 

- Setting Preamble -

To set the preamble text use setParameterByReference with BCD_PARAM_PREAMBLE_STRING as first argument.

The second argument of the setParameterByReference method must be a pointer to a 4-byte length field followed by a Unicode character array (2 bytes per character). The length field must contain the number of characters and not the number of bytes of the Unicode character array. The Unicode character array must contain the string the user wants to set as preamble.

For example, to set L”viperce” call:

 

SetParameterByReference(BCD_PARAM_PREAMBLE_STRING, Pointer);

- Getting Preamble -

To get the preamble text use getParameterByReference with BCD_PARAM_PREAMBLE_STRING as first argument.

The second argument of the getParameterByReference method must be a pointer to a 4-byte length field followed by an empty Unicode character array (2 bytes per character). The length field must be the capacity in terms of characters, and not bytes, of the following Unicode array. The method getParameterByReference copies the preamble text into the Unicode array, only if its length is ≤ than the length contained by the 4-byte length filed, while it always copies the preamble text length into the 4-byte length field.

If the user needs to know the preamble text length (without copying it), the length field must be set to 0 when calling the getParameterByReference method.

 

For further details, see the preamble/postamble examples.

 

 

NOTE

Note that the preamble/postamble examples use data structures to facilitate getting/setting a preamble/postamble, while the samples available in the installation tree directory use classes.

 

 

Syntax

Name:

BCD_PARAM_PREAMBLE_STRING

Set by:

Reference

Values:

see above

Default:

empty string

 

 

Requirements

Header:

declared in BCD_PARAMETERS.H, include SE_PARAMETERS.H.

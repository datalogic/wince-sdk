DecodePostPressMsg

 

These messages occur at a system level of visibility, and are specific to the decode manager ("DCD0:").  Only one message can be in effect at any time.  

Multiple triggers exist on a device, but only one trigger can be active at time.  Once a scanning session begins, all triggers except the one which started the session will be ignored until the end of the scanning session.

 

BOOL DecodePostPressMsg(HWND hWnd, UINT uMsg);

 

Parameters:

HWND hWnd

Handle to a window to receive a message when a trigger is pressed.  If both hWnd and uMsg are set to NULL, the notification is cancelled (see Remarks section).

UINT uMsg

to send to the window when a trigger is pressed.

Returns:

A value of TRUE is returned on success, otherwise FALSE is returned to indicate failure.

 

Remarks:

A message notification cannot be requested if an event has already been requested.  Setting both hWnd and uMsg to NULL can be used to cancel a previous message notification request.

DecodeCancelNotifyRelease

 

The notifications are posted to the decode manager, so the notifications apply to all decoding triggers on the device.  If a trigger has been configured as a keypress, it will not generate release events.  Notifications are sent when the trigger is released.
Multiple triggers exist on a device, but only one trigger can be active at time.  Once a scanning session begins, all triggers except the one which started the session will be ignored until the end of the scanning session.

 

BOOL DecodeCancelNotifyRelease(void);

 

Parameters:

None

 

Returns:

A value of TRUE is returned on success, otherwise FALSE is returned to indicate failure.

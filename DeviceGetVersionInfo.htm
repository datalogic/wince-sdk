DeviceGetVersionInfo

 

This function retrieves the version information associated with the index.  The number of  system/sub-system versions can be found by first calling DeviceGetVersionCount()

 

BOOL cdecl DeviceGetVersionInfo(DWORD index, WCHAR *label, int labelLen, WCHAR *version,

int versionLen);

 

 

Parameters

DWORD index

WCHAR * label

Pointer to a buffer that will receive the version label. The buffer should be at least VERSION_LABEL_SIZE (see section Minimum Input Array Lengths) WCHARs in length. The string returned will be null terminated.

int labelLen

Length, in characters, of the buffer allocated for the label.

WCHAR * version

Pointer to a buffer that will receive the firmware version number. The buffer should be at least VERSION_SIZE (see section Minimum Input Array Lengths) WCHARs in length. The string returned will be null terminated.

int versionLen

Length, in characters, of the buffer allocated for version.

Return Value

A value of TRUE is returned on success, otherwise FALSE is returned to indicate failure.

 

Remarks

Valid indexes range from 0 to one less than the value returned by DeviceGetVersionCount( ).  See Version Index Definitions in Defined Values.  The firmware version index is defined, because the firmware version is always the first version returned in the array.  The presence of other drivers may be device dependent, so the exact number and order of versions reported after the firmware version is not guaranteed.
If successful, then the buffers pointed to by label and version contain the corresponding strings, if failed then label and version contain empty strings (the first element is the null termination character).

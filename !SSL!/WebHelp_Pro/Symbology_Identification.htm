Symbology Identification

Datalogic provides multiple methods for communicating the type of symbology that was used to encode the decoded data.  These methods include returning numeric values which can be compared to defined values representing the various symbologies.  Alternatively, characters identifying the symbology can be added to the decoded data that is returned.

 

Numeric Identification

The Datalogic SDK returns numeric values identifying the symbology of the decoded data separately from the decoded data.  Two numeric values are returned, a code identifier and a code identifier modifier.  These values can be used to differentiate between different symbologies and details within a symbology, respectively.  For values representing different symbologies, see the CODE_ID enumeration in the C/C++ SDK, the CodeId enumeration in the .NET SDK, and the APIEnum class in the Java SDK for language specific implementations.

The code identifier modifiers that are returned are as specified by the AIM USA Symbology Identifier Guideline.  This is the same value that is reported as a hex encoded digit at the end of the AIM Symbology Identifier (which can be returned as part of the decoded data, see below).  An example of how this might be used is in differentiating between Code 128 and the symbology that is a subset of Code 128, GS1-128.  Both of these symbologies would have the same Code ID of 8.  The Code ID Modifier is normally equal to 0, but the value of 1 indicates that the data was encoded using the GS1-128.  

Each symbology has its own definition for the meanings of the Code ID Modifier; please reference the individual specifications for further details, and the CODE_ID Modifiers enumeration in the C/C++ SDK, the Enumerations List (CodeIdModXXX enumerations) in the .NET SDK, and the APIEnum class in the Java SDK for language specific implementations.

 

Identifier Returned with the Decoded Data

Alternatively, a Symbology Identifier can be returned as part of the decoded data.  The decoding parameter named “Send Symbology Identifiers” is used to configure how the Symbology Identifier is inserted into the decoded data.  Options include using Datalogic Label Identifiers, AIM identifiers (see the AIM USA Symbology Identifier Guideline), or User Defined identifiers, either as a prefix or postfix to the label data. 

For a complete description of options, see the “Send Symbology Identifiers” parameter under General Decoding Definitions.

DecodePostReleaseEvent

 

When the trigger that initiated the press event is released the software must send a release event regardless of configured type. This event will be set before the events of any resultant events (ie scan stop.

These events occur at a system level of visibility, and are specific to the decode manager ("DCD0:").  Only one event can be in effect at any time.  If a release event is posted, then you need to cancel the stop notification to post the release event with a new event handle or a windows message.

Multiple triggers exist on a device, but only one trigger can be active at time.  Once a scanning session begins, all triggers except the one which started the session will be ignored until the end of the scanning session.

 

BOOL DecodePostReleaseEvent(HANDLE hEvent);

 

Parameters:

HANDLE hEvent

Handle to the event to signal when a trigger is released.  If set to NULL, the notification is cancelled (see Remarks section).

Returns:

A value of TRUE is returned on success, otherwise FALSE is returned to indicate failure.

 

Remarks:

An event notification cannot be requested if a message has already been requested. Setting hEvent to NULL can be used to cancel a previous event notification request.

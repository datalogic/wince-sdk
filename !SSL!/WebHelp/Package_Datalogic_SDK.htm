Package Datalogic.SDK Description

Provides the classes necessary to access terminal based functionality. This SDK is a wrapper around the eVC++ SDK created for Datalogic devices.

 

The Datalogic.SDK package is divided into four basic entities:

  • DecodeHandle provides access to a specific decoding device. This also includes capability to listen for decoder related events.

  • DeviceAPI provides access to terminal related functionality.

  • DecodeAPI provides access to terminal decoding related functionality.

  • Param provides access to get and set values on the terminal.

 

Developing/Deploying applications to the Terminal

For Development:

  • Use import Datalogic.SDK.*; in any class that requires access to Datalogic.SDK functionality.

  • When compiling your classes, be sure the DatalogicSDK.jar file is included in the class path.

  • Constant values are stored in APIEnum. (example: APIEnum.DCD_BARCODE; will return the APIEnum.DCD_BARCODE value)

For Deployment:

  • Applications must be bundled into a jar file to be deployed/run on a terminal.

  • In the command line for the evm on the terminal, be sure to include the full path to DatalogicSDK.jar as part of the class path. The following example shortcut would run the app class within the SampleApps.jar file with access to the Datalogic.SDK package.

"\windows\evm.exe" -cp \windows\DatalogicSDK.jar;\app\app.jar app

Note: By default, the DatalogicSDK.jar file is placed in the \Windows folder on every terminal.

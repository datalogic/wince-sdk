Platform Compatibility with Pocket PC

 

 

The following libraries have been included in the Datalogic mobile computer software image to support applications developed for Pocket PC.

 

 

File

Note

aygshell.dll

By default all Windows CE-based platforms with display support the functionality of the standard shell. From Windows CE.NET 4.2 version on, OEMs can include API compatibility support for the Pocket PC2002 shell in their Windows CE-based platforms. These extensions to the standard shell are known as the AYGShell APIs. AYGShell support allows most of Pocket PC2002-based applications to run on the Windows CE device after being recompiled for an OEMâ€™s Windows CE-based platform.

Doclist.dll

Extension of MFC classes for Pocket PC.

note_prj.dll

Extension of MFC classes for Pocket PC.

mfcce300.dll

Necessary for compatibility with applications using MFC3 instead of MFC4.

gx.dll

Proprietary implementation of GAPI 1.2 for streaming, games, multimedia. This API is required for example for RealPlayer and for other applications using in a huge way graphic on LCD.

bthutil.dll

Necessary for compatibility with Pocket PC platform.

SDCConfig

Standard connection configuration struct

 

 

Members:

 

char configName[CONFIG_NAME_SZ]

"Friendly" name to associate with this connection

char SSID[SSID_SZ]

SSID of the network to connect to

AUTH authType

Authentication type

ENCRYPTION encr

Encryption type
UNION

CRYPT PSK

Pre-shared (secret) key value

WEP_KEY wep

WEP key value and index

EapTypes EapType

EAP type of the configuration. Only valid for WPA and WPA2 profiles. Important: You must use the corresponding union type with the value set in the EapType field:

 

  SDCConfig sdc {};

  sdc.EapType = EapTypes.PPP_EAP_PEAP;

  

  PEAPConfig peapConfig {};

  sdc.PEAP = &peapConfig;

 

BOOL PmkCache

EAP: Use Pairwise Master Key (PMK) caching?

Allows authenticating clients can skip EAP exchange while roaming between access points.

1 (TRUE) = yes, use PMK caching

0 (FALSE) = no, don't use PMK caching

BOOL Cckm

EAP: Use Cisco Centralized Key Management (CCKM) Fast Roaming?

1 (TRUE) = yes

0 (FALSE) = no

PEAPConfig *PEAP

Use *one* of the values in the union to point to a struct of the desired EAP method type. All values in the given struct should be initialized to 0.

 

The recommended methods to do so are as follows:

 

Option 1: use memset

  PEAPConfig peapConfig;

 memset(peapConfig, 0, sizeof(peapConfig));

 

Option 2: Use C++ initialization

  PEAPConfig peapConfig {};

Class DecodeHandle

 

public class DecodeHandle

extends java.lang.Object

 

DecodeHandle is used to access a decoding device on the terminal. DecodeHandle also allows for decoding device event listening and other decoder related functionality.

 

Using the DecodeHandle instance to post requests using events:

 

  1. Call DecodeHandle.createEvent method to get a new event ID.

  2. Call DecodeHandle.addActionListener specifying the event ID and a DecodeActionListener object.

  3. Call DecodeHandle.postRequestEvent specifying the event ID and the type of request.

 

When the listener within DecodeHandle is used for posting requests using events, the application should call DecodeHandle.removeActionListeners before it is closed.

 

If the listener is not closed correctly, the garbage collector will automatically call DecodeHandle.removeActionListeners. One indicator that the listener was not shut down correctly is that the console will remain open for a number of seconds after the application exits

.

Note: The methods DecodeHandle.addActionListener and DecodeHandle.postRequestEvent, that do not require an event id parameter, are maintained for backwards compatibility.

DecodePostPressEvent

 

When all hardware triggers are released and a hardware trigger is pressed the software must sent a press event regardless of the configured type of the trigger.  This event will be set before the events of any resultant events (ie scan start).
These events occur at a system level of visibility, and are specific to the decode manager ("DCD0:").  Only one event can be in effect at any time.  If a press event is posted, then you need to cancel the stop notification to post the press event with a new event handle or a windows message.
Multiple triggers exist on a device, but only one trigger can be active at time.  Once a scanning session begins, all triggers except the one which started the session will be ignored until the end of the scanning session.

 

BOOL DecodePostPressEvent(HANDLE hEvent);

 

Parameters:

HANDLE hEvent

Handle to the event to signal when a trigger is pressed.  If set to NULL, the notification is cancelled (see Remarks section).

Returns:

A value of TRUE is returned on success, otherwise FALSE is returned to indicate failure.

 

Remarks:

An event notification cannot be requested if a message has already been requested.  Setting hEvent to NULL can be used to cancel a previous event notification request.

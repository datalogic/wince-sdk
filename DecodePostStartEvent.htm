DecodePostStartEvent

 

These events occur at a system level of visibility, but are separate for each decoding device. For each decoding device, only one event can be in effect at any time. If a start event is posted, then you need to cancel the start notification to post the start event with a new event handle or a windows message.

 

BOOL DecodePostStartEvent(HANDLE hFile, HANDLE hEvent);

 

Parameters:

HANDLE hFile

Handle to the decoding device from which a scanning start notification is requested.  The CreateFile function returns this handle.

HANDLE hEvent

Handle to the event to signal when scanning starts.  If set to NULL, the notification is cancelled (see Remarks section).

 

Returns:

A value of TRUE is returned on success, otherwise FALSE is returned to indicate failure.

 

Remarks:

An event notification cannot be requested if a message has already been requested.  Setting hEvent to NULL can be used to cancel a previous event notification request, but only if the same file handle is used to both request and cancel the notification.
